**产品需求文档：锦绣非遗数字展览馆**

**一、产品概述**

**产品名称**：锦绣非遗——指尖上的东方美学

**核心理念**：以“指尖触碰历史，科技点亮传承”为核心，利用Web前端技术构建一个集视觉欣赏、互动体验、知识科普于一体的轻量级非遗数字展馆。

**技术栈约束**：HTML5, CSS3, JavaScript。

**目标用户**：对传统文化感兴趣的大学生、年轻群体、寻找教学素材的教育工作者。

**二、UI设计与视觉规范**

符合“简约、养眼、柔和”且不失文化底蕴的要求，采用“新中式极简风”。

**主色调**：

**月白**：作为页面背景色，替代纯白，减少视觉疲劳，营造宣纸般的质感。

**朱砂**：作为强调色（按钮、图标、重点文字），用于点缀，象征非遗的生命力。

**辅助色**：

**黛蓝**：用于导航栏和页脚，沉稳大气，与朱砂形成冷暖对比。

**竹青**：用于成功状态提示或次要装饰，增加自然气息。

**字体与排版**：

标题使用楷体，正文使用黑体以保证阅读清晰度。

大量使用留白，卡片式设计，圆角处理，配合轻微的投影增加层次感。

本项目规划为 **5个核心页面**，采用“单页应用”式的平滑滚动体验或标准的多页面跳转结构。

**四、功能清单与详细描述**

本部分详细拆解功能，重点描述前端实现逻辑与修饰效果。

**1\. 全局通用功能**

**导航栏**：

**形式**：固定在顶部，背景半透明磨砂效果。

**修饰**：鼠标悬停在菜单项上时，文字下方出现一条朱砂色的下划线，从左至右滑入动画。

**2\. 首页**

**沉浸式Banner**：

**形式**：全屏宽度的轮播图，展示高清非遗大图（如刺绣特写）。

**修饰**：图片加载时有淡入效果；文字标题带有打字机效果。

**非遗卡片**：

**形式**：图文混排卡片。

**修饰**：鼠标悬停时，卡片轻微上浮并加深阴影，图片轻微放大。

**3\. 博览页**

**动态筛选**：

**形式**：点击“剪纸”、“皮影”等按钮，下方内容区域刷新。

**详情模态框**：

**形式**：点击卡片后，屏幕中央弹出半透明遮罩层，显示详细内容。

**修饰**：弹出时带有缩放弹跳效果。

**4\. 核心功能一：非遗拼图小游戏**

**游戏主题**：“修复国宝”

**具体描述**：一款基于Canvas的图片拼图游戏

**功能描述**：

系统将一张非遗高清图片（如青花瓷瓶）切割成 3x3 或 4x4 的九宫格，其中留空一格，玩家需通过点击相邻方块移动位置，最终还原图片。

**具体实现细节**：

**界面布局**：左侧为游戏画布，右侧为“原图预览”、“步数统计”、“计时器”和“难度选择”按钮。

**交互逻辑**：

**打乱**：点击“开始”，JS算法随机移动方块50次，确保有解。

**移动**：监听画布点击事件，计算点击坐标与空白格的相邻关系，若相邻则交换位置。

**胜利判定**：每次移动后，JS校验当前数组顺序是否与原始顺序一致。

**修饰与反馈**：

**胜利特效**：当拼图完成，画布上方飘落彩带（使用简单的JS粒子系统或CSS动画）。

**胜利弹窗**：显示“修复成功！耗时XX秒”。

**5\. 核心功能二：AI非遗小助手**

**功能描述**：一个基于大模型接口的智能问答机器人，用于解答用户关于非遗历史、制作工艺的问题。

**具体实现细节**：

**入口**：页面右下角悬浮的“黛蓝”色圆形按钮，图标为机器人。

**交互形式**：

点击按钮，展开一个聊天窗口。

用户输入框支持回车发送。

聊天记录以气泡形式展示，用户为右侧绿色气泡，AI为左侧白色气泡。

**技术实现：**

**前端逻辑：**使用 fetch 或 XMLHttpRequest 发送POST请求。

**接口对接：**对接通义千问或其他开源LLM的API。

**流式输出模拟：**为了体验更好，JS接收到返回文本后，使用打字机效果逐字显示，模拟AI思考过程。

**预设问答**：如果用户不知道问什么，聊天窗口顶部提供“猜你想问”的快捷标签（如：“剪纸的起源？”“皮影戏怎么做？”），点击自动填入输入框。
