# 锦绣非遗数字展览馆 PRD

## 1. 产品概述

- 产品名称：锦绣非遗
- 副标题：指尖上的东方美学
- 产品定位：基于 HTML5、CSS3、JavaScript 的纯前端非遗数字展览馆课程作品
- 目标用户：
  - 对传统文化感兴趣的大学生
  - 需要课堂展示素材的教师与同学
  - 希望快速了解非遗项目的普通访客
- 核心目标：
  - 用轻量多页面网站完成非遗内容展示
  - 覆盖课程评分项中的页面结构、样式设计、交互功能、Canvas 绘制和特色功能
  - 保持基础前端作业气质，不引入框架和复杂工程结构

## 2. 实现约束

本项目执行自定义 vibe skill `front-homework-vibe` 的约束，核心规则如下：

- 只使用 HTML、CSS、原生 JavaScript
- 公共布局和通用样式统一抽到共享 CSS 中
- 命名简短、直观，避免工程化和 AI 味很重的超长命名
- 保留必要 Canvas 功能，但不把整站过度改造成 Canvas 项目
- 小交互优先用 DOM + CSS + 原生事件完成

本次版本另外执行的用户定制要求：

- 删除“绘制纹样”模式
- 保留“戏曲脸谱绘制”画板
- 脸谱画板新增颜色选择和画笔粗细控制
- 视觉方向采用“新中式极简风”，以月白、朱砂、黛蓝、竹青为主色

## 3. 信息架构

虽然产品叙述采用“首页 / 博览 / 工坊 / 智库 / 关于”五大语义区，但为了满足课程评分项，项目实际保留以下页面结构：

- 登录页：`login.html`
- 首页：`index.html`
- 分类页：
  - `category-tradition.html`
  - `category-folk.html`

- 详情页：
  - `detail-celadon.html`
  - `detail-xuanzhi.html`
  - `detail-blueprint.html`
  - `detail-spring-festival.html`
  - `detail-dragon-boat.html`
  - `detail-solar-terms.html`
  - `detail-kunqu.html`
  - `detail-jingju.html`
  - `detail-yueju.html`
  - `detail-paper-cut.html`
  - `detail-suxiu.html`

语义映射关系如下：

- 关于页：`contact.html`

## 4. 页面需求

### 4.1 登录页

- 提供用户名和密码输入框
- 前端校验用户名非空、密码长度合法
- 登录成功后把用户名保存到 `localStorage`
- 登录后首页、分类页、详情页、关于页顶部展示当前用户名

### 4.2 首页

首页是整站核心入口，必须包含：

- 固定顶部导航
- 沉浸式开篇长幕
- 四馆入口
- 推荐非遗概览
- 视频推荐区
- Canvas 脸谱工坊
- 问答小助手
- 拖放收藏馆
- 右侧浮动导览
- 页脚

首页重点交互：

- 轮播图支持左右切换和圆点切换
- 标题使用打字机效果
- “进入展馆”支持平滑滚动到分类入口区
- 右侧导览支持快速跳转到首页重点区块
- 推荐展品支持拖入收藏馆

### 4.3 分类页

分类页统一承担“博览页”职责，每个分类页都需要：

- 顶部分类说明区
- 中部卡片列表区
- 分页按钮
- 右侧导览侧栏
- 迷你收藏馆

分类页重点交互：

- 通过分页按钮切换当前列表内容
- 点击“快速预览”打开模态框
- 模态框中展示摘要、历史、工艺、价值和视频入口
- 点击“查看详情”跳转到独立详情页
- 卡片支持拖入迷你收藏馆

### 4.4 详情页

每个非遗项目详情页包含：

- 项目标题与分类信息
- 历史渊源
- 工艺特点
- 传承价值
- 视频演示入口
- 相关项目推荐

详情页目标：

- 作为老师检查“详情展示子页面”的直接页面证据
- 作为分类页快速预览后的深入阅读页

### 4.5 关于页

关于页复用 `contact.html`，包含：

- 项目背景
- 联系信息
- 团队介绍
- 留言板表单
- 最近留言展示

留言板要求：

- 姓名、邮箱、手机号、留言内容做前端校验
- 提交后保存到 `localStorage`
- 页面刷新后可继续看到最近留言

## 6. 视觉规范

### 6.1 风格方向

- 风格关键词：新中式极简、柔和、清爽、宣纸质感
- 页面避免过重的暗色和工业化风格
- 页面留白充足，卡片圆角适中，阴影轻

### 6.2 配色

- 月白：页面背景主色
- 朱砂：按钮、强调内容、重点标题
- 黛蓝：导航、辅助强调、深色模块
- 竹青：标签、成功态、局部装饰

### 6.3 字体

- 标题：楷体 / 宋体风格
- 正文：系统中文无衬线字体

## 7. 交互功能需求

### 7.1 公共交互

- 当前用户名状态显示
- 顶部导航高亮
- 退出登录
- 收藏数量角标同步更新

### 7.2 轮播与动效

- 首页轮播图自动播放
- 支持手动切换
- 标题打字机效果
- 卡片悬浮上浮

### 7.3 拖放收藏馆

- 首页精选卡片支持拖放
- 分类页卡片支持拖放
- 收藏数据保存在 `localStorage`
- 重复拖入同一项目时不重复添加

### 7.4 Canvas 脸谱工坊

Canvas 区只保留“戏曲脸谱绘制”功能，不再保留纹样绘制。

功能要求：

- 预绘制脸谱底稿
- 提供颜色选择按钮
- 提供画笔粗细范围条
- 支持鼠标按下拖动绘制
- 支持一键清空并恢复底稿

### 7.5 问答小助手

当前版本以馆内常见问题为核心，采用前端问答库完成快速回复。

功能要求：

- 首页内嵌问答小助手面板
- 提供“猜你想问”快捷按钮
- 支持用户输入后发送
- 通过本地关键词匹配返回答案
- 用打字机效果模拟 AI 输出

### 7.6 模态框预览

- 分类页点击“快速预览”后弹出居中模态框
- 支持点击遮罩关闭、关闭按钮关闭、Esc 关闭
- 模态框内包含视频入口和详情页入口

## 8. 数据存储方案

本项目全部使用前端本地存储：

- 用户名：`heritage.username`
- 收藏馆：`heritage.collection`
- 留言板：`heritage.messages`

不依赖数据库和后端服务。

## 9. 技术实现

- 结构层：HTML5
- 样式层：CSS3
- 行为层：原生 JavaScript
- 数据组织：本地 JS 数据文件
- 本地存储：`localStorage`
- 图形绘制：`canvas`

JS 文件职责拆分：

- `assets/js/common.js`：公共状态、导航高亮、用户名和收藏角标
- `assets/js/data.js`：非遗项目数据
- `assets/js/slider.js`：首页轮播和入口区块渲染
- `assets/js/drag.js`：右侧导览、拖放收藏
- `assets/js/canvas.js`：脸谱工坊
- `assets/js/assistant.js`：问答小助手
- `assets/js/pagination.js`：分类页分页、快速预览、详情页渲染
- `assets/js/login.js`：登录校验
- `assets/js/contact.js`：留言板校验和渲染
